Variables

A variable in programming is how we can store and reference information. They're similar to variables in math, except they can store many different types of information instead of just mathematical objects. You can assign information to a variable in one place, and then use that variable in another to retrieve that information.

A variable consists of two parts: a name (called the identifier ) and the information (called the value ).

Types of Information

There are many types of information in programming. This week we focus on using numbers—like 1, 2.5, 3.99, -1, etc—and text, like "Hello World".

Later in the course we'll refine what we mean by numbers and text.

Other arithmetic operators

In addition to the four basic operators, we can use the integer division operator and the modulus operator.

Integer division

The integer division operator (denoted //) is division where the remainder is discarded, i.e. the integer number of times one number goes into another.

For example,

10 // 2 == 5

Since 2 goes into 10 five times. However,

10 // 3 == 3

Since 3 only goes into 10 three times before we need a decimal. If the second number is bigger than the first, then

10 // 11 == 0

Since 11 does not divide 10 a whole number of times at all.

Modulus operator

The modulus operator (denoted %) is the remainder operator. The modulus returns what's left over when you divide one number by another.

note: here we discuss only positive integers with the modulus operator.

For example,

10 % 2 == 0

Since 2 divides 10 with no remainder leftover. However,

10 % 3 == 1

Since 3 divides 10 with 1 left over. If the second number is greater than the first then

10 % 11 == 10

Since 11 does not divide 10 at all, thus 10 is left over.

Truth Values and Relational Operators

In addition to the number and text types of values we've discussed so far, there are boolean types that can take one of two values. In this case, we have true and false.

A relational operator examines the relationship between two values and returns whether than relation is true or false.

For example, Trinket blocks (and Python) defines equals, not equals, less than, less than or equals to, greater than, and greater than or equals to.

In future weeks we'll discuss the rules for combining truth values in different ways (important to the discrete math portion of the certification exam)!
